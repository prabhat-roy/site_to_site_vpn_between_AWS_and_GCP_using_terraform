# Site-to-Site VPN: AWS ↔ GCP (Terraform)

IPsec site-to-site tunnel between an AWS VPC and a GCP VPC, with a test EC2 + GCE VM at each end for connectivity verification.

---

## Architecture

```
AWS VPC (10.0.0.0/16)               GCP VPC (10.10.0.0/24)
   |                                       |
   +-- EC2 (10.0.x.x) <-- IPsec VPN -->   +-- VM (10.10.x.x)
   |                                       |
AWS VGW + 2 tunnels                   GCP HA VPN Gateway + 2 tunnels
```

Pre-shared keys are generated by Terraform; HA tunnels with BGP-style failover via static routes.

---

## What gets created

| Side | Resources | File |
|------|-----------|------|
| AWS  | VPC, subnets, route tables, IGW, Customer Gateway, VPN Connection, VGW, EC2 | `aws.tf`, `aws_ec2.tf` |
| GCP  | VPC, subnet, firewall, HA VPN Gateway, Peer VPN Gateway, VPN tunnels, GCE VM | `gcp.tf`, `gcp_vm.tf` |
| Outputs | Tunnel IPs, instance IPs | `output.tf` |

---

## Prerequisites

1. AWS Administrator credential via `aws configure`.
2. GCP service account with `Compute Admin`; JSON key path in `terraform.tfvars`.
3. EC2 key pair (AWS) + SSH key (GCP).
4. Terraform >= 1.5.

---

## Configuration

`terraform.tfvars` — set AWS region/CIDR/AMI/key + GCP project/region/credentials/keys.

---

## Deploy

```bash
terraform init
terraform plan
terraform apply -auto-approve
```

---

## Verify

SSH to either VM, ping the other side's private IP. Round-trip should be < 80ms in same Asia region; check tunnel status in AWS VPN console + GCP HA VPN dashboard.

---

## Tear down

```bash
terraform destroy -auto-approve
```
